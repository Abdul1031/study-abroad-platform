# Deployment guide ‚Äî free tier

Three services, all with a permanent free tier and no credit card required:

| Layer       | Service                      | Why                                                                                        |
| ----------- | ---------------------------- | ------------------------------------------------------------------------------------------ |
| Database    | [Neon](https://neon.tech)    | Serverless Postgres, free tier has no expiry (unlike Render's 30-day Postgres trial)       |
| Backend API | [Render](https://render.com) | Free web service; sleeps after 15 min idle, wakes on the next request (~30‚Äì50s cold start) |
| Frontend    | [Vercel](https://vercel.com) | Free static hosting, instant global CDN, auto-deploys on push                              |

Total time: ~15 minutes. All three steps below require signing in with your own GitHub account (Abdul1031) ‚Äî that OAuth consent has to happen in your browser, not through a CLI or agent.

## 1. Database ‚Äî Neon

1. Go to [neon.tech](https://neon.tech) ‚Üí sign up with GitHub.
2. Create a project (any name, region closest to your Render region ‚Äî pick `Frankfurt (eu-central-1)` to match the Render config below).
3. Copy the **connection string** from the dashboard (it looks like `postgresql://user:pass@ep-xxxx.eu-central-1.aws.neon.tech/neondb?sslmode=require`). Keep this tab open ‚Äî you'll paste it into Render next.

## 2. Backend ‚Äî Render

1. Go to [render.com](https://render.com) ‚Üí sign up with GitHub ‚Üí authorize access to this repository.
2. **New ‚Üí Blueprint** ‚Üí select the `study-abroad-platform` repo. Render detects [`render.yaml`](./render.yaml) automatically and proposes the `study-abroad-api` web service.
3. Click **Apply** ‚Äî the first deploy will fail (missing `DATABASE_URL`), that's expected.
4. Open the new service ‚Üí **Environment** tab ‚Üí set:
   - `DATABASE_URL` ‚Üí the Neon connection string from step 1
   - `ADMIN_EMAILS` ‚Üí your email, comma-separated if more than one (grants access to `/admin`)
   - `FRONTEND_URL` ‚Üí leave as `http://localhost:3000` for now; you'll update it after step 3
   - `JWT_SECRET` is already auto-generated by the blueprint ‚Äî leave it
5. **Manual Deploy ‚Üí Deploy latest commit**. Watch the logs: `npm ci` ‚Üí `prisma generate` ‚Üí `tsc build` ‚Üí `prisma migrate deploy` ‚Üí server starts.
6. Once live, note the URL: `https://study-abroad-api.onrender.com`. Confirm it works:
   ```bash
   curl https://study-abroad-api.onrender.com/api/health
   ```
7. Seed the university catalog (one-time, from your machine ‚Äî Render's free plan has no shell access, so run it locally against the Neon DB):
   ```bash
   cd backend
   DATABASE_URL="<paste Neon connection string>" npx tsx prisma/seed_universities_de.ts
   ```

## 3. Frontend ‚Äî Vercel

1. Go to [vercel.com](https://vercel.com) ‚Üí sign up with GitHub ‚Üí **Add New ‚Üí Project** ‚Üí import this repo.
2. Vercel needs to build only the `frontend` workspace:
   - **Root Directory**: `frontend`
   - **Framework Preset**: Vite (auto-detected)
   - Build/output settings are read from [`frontend/vercel.json`](./frontend/vercel.json) ‚Äî no changes needed.
3. **Environment Variables** ‚Üí add `VITE_API_URL` = `https://study-abroad-api.onrender.com/api` (your Render URL from step 2, with `/api`).
4. **Deploy**. Vercel gives you a URL like `https://study-abroad-platform.vercel.app`.

## 4. Close the loop ‚Äî CORS

Go back to Render ‚Üí your service ‚Üí **Environment** ‚Üí set `FRONTEND_URL` to your Vercel URL (no trailing slash), e.g. `https://study-abroad-platform.vercel.app`. Save ‚Äî Render redeploys automatically. Without this step the API rejects requests from the deployed frontend (CORS + the origin allowlist in `security.middleware.ts`).

## Verifying the full deploy

1. Open your Vercel URL, sign up as a test student.
2. Complete the profile wizard.
3. Check `/recommendations` ‚Äî should show ranked universities.
4. Log in with the email you put in `ADMIN_EMAILS` and visit `/admin` ‚Äî should show the quality dashboard.

## Notes on the free tier

- **Render cold starts**: the free web service sleeps after 15 minutes of no traffic. The first request after sleep takes 30‚Äì50 seconds while it wakes up ‚Äî this is expected, not a bug. A cron-based uptime pinger (e.g. a free [cron-job.org](https://cron-job.org) hitting `/api/health` every 10 min) keeps it warm if that matters for a demo.
- **Neon autosuspend**: similarly, Neon's compute suspends after inactivity and resumes on the next query with a brief delay ‚Äî no action needed, Prisma's connection retry handles it.
- **Scraper cron**: the weekly Sunday 02:00 UTC scrape only fires while the Render service is awake. On the free tier this is unreliable ‚Äî use the `POST /api/scraper/run` admin endpoint (or the Admin dashboard's "Run full sync now" button) to trigger scrapes on demand instead.
